Текущий проект.txt

Проект базы для системы агрегации сайтов вакансий.
1) Бизнес требования: 
1.1) Поддержка работы разных сайтов вакансий (hh.ru и др)
1.2) Посчет вакансий по ключевым словам
1.3) При ошибках выполнения должно быть логирование в таблицы sql характера ошибок

2) Системные требования:
2.1) База поддерживает репликацию master-slave, требуется подготовить скрипты создания публикации/подписчиков
2.2) База должна быть настроена в режиме зеркалирования

Возможный проект №2(version_0.1).txt
-----------------------------------------------------------------
Проект базы для системы регистрации событий разных источников (пока думаю каких, буду рад подсказке, может агрегация существующих систем открытых) с упором на базы данных.
Для создания потока событий в БД требуется создать(использовать готовый) генератор событий 3 разных типов для источника.
И программу отображения поступления событий в виде сетки источник/время/тип события.
Язык для прикладного ПО - c#. 

Требования по курсу ms server:
Ядро MS SQL, опциональное включение очереди service broker или rabbitmq через ПО или флагом в хранимой процедуре.
Сделать упор на не блокирующие чтение/запись, в трассе ms sql при запуске эмулятора/программы отображения должны отсутствовать эскалации таблицы при обработке от 100 000 записей.

База поддерживает транзакционную репликацию типа master-slave, подготовить скрипт создания публикации базы и подписки.
